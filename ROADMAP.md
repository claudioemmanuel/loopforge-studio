# Loopforge Studio Roadmap

This document outlines the planned features and improvements for Loopforge Studio. We build in the open and welcome community input on priorities.

## How We Prioritize

We focus on features that:
1. **Improve reliability** - Make the AI coding workflow more dependable
2. **Increase accessibility** - Support more AI providers and deployment options
3. **Enhance visibility** - Help users understand what the AI is doing
4. **Enable collaboration** - Support team workflows

---

## Current Focus (Q1 2025)

These are actively being worked on or planned for the near term.

### Local AI Support
- [ ] **Ollama integration** - Run local models for privacy-conscious users
- [ ] **Custom model endpoints** - Support any OpenAI-compatible API

### Improved Execution Visibility
- [ ] **Diff previews** - See proposed changes before AI commits
- [ ] **Rollback support** - Easily undo AI-generated commits
- [ ] **Execution timeline** - Visual history of all AI actions

### Developer Experience
- [ ] **Keyboard shortcuts** - Power user navigation
- [ ] **Task templates** - Save common task patterns
- [ ] **Better error messages** - Actionable guidance when things go wrong

---

## Next Up (Q2 2025)

Features planned after the current focus areas.

### Team Features
- [ ] **Workspaces** - Shared repositories across team members
- [ ] **Role-based access** - Admin, developer, viewer permissions
- [ ] **Activity feed** - See what your team is working on

### Enhanced AI Capabilities
- [ ] **Multi-file awareness** - Better understanding of project structure
- [ ] **Test execution** - Run tests before committing changes
- [ ] **PR creation** - Automatically create pull requests with summaries

### Integrations
- [ ] **GitHub Issues sync** - Import issues as tasks
- [ ] **Webhooks** - Notify external systems of task events
- [ ] **Slack notifications** - Get updates in your team chat

---

## Future Exploration

Ideas we're considering but haven't committed to yet.

### Advanced Workflows
- [ ] **Multi-agent collaboration** - Multiple AIs working on related tasks
- [ ] **Human-in-the-loop checkpoints** - Approval gates during execution
- [ ] **Scheduled tasks** - Run tasks on a schedule (e.g., weekly dependency updates)

### Enterprise Features
- [ ] **SSO/SAML authentication** - Enterprise identity providers
- [ ] **Audit logging** - Detailed logs for compliance
- [ ] **Self-hosted registry** - Air-gapped deployment support

### Platform Expansion
- [ ] **VS Code extension** - Create tasks from your editor
- [ ] **CLI tool** - Create and manage tasks from terminal
- [ ] **Mobile app** - Monitor tasks on the go

---

## Request a Feature

Have an idea that's not on this list?

1. **Check existing issues** - Someone may have already suggested it
2. **Open a discussion** - Share your use case in [GitHub Discussions](https://github.com/claudioemmanuel/loopforge-studio/discussions)
3. **Create an issue** - For well-defined feature requests, [open an issue](https://github.com/claudioemmanuel/loopforge-studio/issues/new?template=feature_request.md)

We review community feedback regularly and update this roadmap based on demand and feasibility.
